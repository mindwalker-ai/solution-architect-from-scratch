# AI, Data & Infrastructure Solution Architect — From Scratch

## Presales Track

**Learn to sell the architecture, not just build it.**

Most technical training teaches you to *operate* a technology. This teaches you to **turn a customer's problem into an architecture, size it, price it, and win the deal.** You start at "what does the business actually need?" and end presenting a full solution — HLD, LLD, BOM, pricing, and a live demo — to an executive board.

53 lessons. 8 phases. ~285 hours (~10–12 months at your own pace). From reading a network diagram to defending a multi-million-dollar AI transformation proposal. Every lesson ships a **real consulting deliverable** — a discovery kit, a reference architecture, a sizing sheet, a BOM, a proposal, a demo script.

You don't just learn cloud, data, and AI. You learn to **architect and sell** them — at *architect altitude*: deep enough to design and defend, without needing to be the implementing engineer.

## How this track is different

| | Typical technical training | This track |
|---|---|---|
| **Starting point** | "Here's the install command" | "What is the customer's problem, and what is it worth?" |
| **Output** | "I deployed it once" | A portfolio of HLDs, BOMs, sizing sheets, and a presales demo |
| **Depth** | Deep engineering internals | *Architect altitude* — design, size, price, defend |
| **Scope** | One technology | Business → Infra → Cloud → Data → AI → Architecture → Presales |
| **Every lesson** | A lab you run | A **deliverable you ship** to `outputs/` |
| **Audience** | Engineers only | Mixed staff — Phase 0 brings anyone up to speed |

## The Lab

Every lab runs **local or on a free tier** — reproducible and near-zero-cost, on any laptop:

- **Local:** Docker, k3s/kind, Proxmox/KVM, MinIO/Ceph, Postgres, DuckDB, Kafka, Ollama, vLLM (CPU/small-GPU), Milvus/Qdrant, Prometheus/Grafana, DevStack, LocalStack.
- **Free tier:** one small VM on AWS/Azure/GCP for the cloud lessons.

Labs exist to **validate a design** — size a cluster on paper, then sanity-check it against a small local benchmark. You are learning to architect, not to run production.

## Getting Started

```bash
git clone https://github.com/mindwalker-ai/solution-architect-from-scratch
cd solution-architect-from-scratch

# Read lessons straight from the repo:
#   phases/<phase>/<lesson>/docs/en.md
# Each lesson ships a deliverable into its outputs/ directory.

# Optional: build the course website locally
node site/build.js && open site/index.html
```

New to IT? Start at **Phase 0 → How Enterprise IT Fits Together**. Already an engineer? Use `/find-your-level` in Claude Code to jump ahead.

## What You Ship

Every lesson saves a reusable deliverable to its `outputs/` directory. By the end you have the **Presales Solution Architect Toolkit**, not just notes:

- **Discovery kit** — questionnaire, stakeholder map, MEDDICC sheet
- **Reference architectures** — on-prem, AWS, Azure, GCP, data platform, AI platform
- **HLD / LLD templates**, **BOM template**, **sizing calculators (incl. GPU)**, **TCO/ROI model**
- **Proposal & executive-summary templates**, battlecards, demo scripts, an executive deck template
- **A portfolio** of four flagship capstones you can defend in an interview or a real deal

## Part of Mindwalker Academy
