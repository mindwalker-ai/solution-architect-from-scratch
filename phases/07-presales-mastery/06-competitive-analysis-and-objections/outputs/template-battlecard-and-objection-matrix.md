## Usage notes

- **This is a living document, not a one-off deliverable.** Refresh it every time the customer's risk register is re-scored (§7's rehearsal record is also where you log the refresh date).
- **Every "Source" cell is mandatory.** A row that can't name a risk-register entry or a specific competitive claim doesn't belong in the matrix — see `06-competitive-analysis-and-objections/docs/en.md` for why an unsourced objection is a guess dressed as intelligence.
- **A worked example for Cakrawala Group is filled in separately** — see [`example-cakrawala-group-battlecard.md`](./example-cakrawala-group-battlecard.md) in this same `outputs/` folder — so this template stays a clean, reusable skeleton a colleague can copy for any deal.
